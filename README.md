# toio Music Program

toioロボットキューブを使った音楽プログラム。複数のtoio（最大3台）をBluetooth接続し、位置に応じたリアルタイム音声合成と動きの記録・再生を行います。

## toioとは

**toio**は、ソニー・インタラクティブエンタテインメントが開発した小型ロボットキューブです。

### 主な特徴

- **サイズ**: 約32mm x 32mm x 19mm の小型キューブ
- **通信**: Bluetooth Low Energy (BLE) でPCやスマートフォンと接続
- **センサー類**:
  - **位置センサー**: 専用マット上でX/Y座標と角度を検出（光学式）
  - **磁気センサー**: 磁石の接近を検出
  - **モーションセンサー**: 傾きや衝突を検出
  - **ボタン**: 上面に1つのタクトスイッチ
- **アクチュエータ**:
  - **モーター**: 2輪駆動で前後移動・回転が可能
  - **LED**: フルカラーLEDで状態表示
  - **スピーカー**: 効果音の再生が可能

### 専用マット

toioは専用のプレイマット上で動作します。マットには特殊なパターンが印刷されており、toioの底面にある光学センサーがこのパターンを読み取ることで、自身の位置（X座標、Y座標）と向き（角度）をリアルタイムで取得できます。

```
マットの座標系:
  +------------------+
  |                  |
  | (45,45)  →X      |
  |    ↓Y           |
  |                  |
  |          (455,455)|
  +------------------+

座標範囲: X: 45〜455, Y: 45〜455
```

## システム概要

本システムは、toioキューブの**位置情報を音に変換**することで、物理的な動きを音楽的な表現に変えるインタラクティブな音楽システムです。

### コンセプト

1. **身体性のある音楽制作**: toioを手で動かすことで、直感的に音を作り出せます
2. **ループベースの作曲**: 動きを記録し、ループ再生することでDTMのような音楽制作が可能です
3. **複数パートの演奏**: 最大3台のtoioを使い、異なる波形（楽器）を同時に演奏できます

### 動作の流れ

```
[ユーザーがtoioを動かす]
        ↓
[位置センサーがX/Y座標を検出] ← 50Hz（20ms間隔）でサンプリング
        ↓
[座標を周波数・音量に変換]
        ↓
[リアルタイムで音声を合成・出力]
        ↓
[動きをフレームとして記録]
        ↓
[ループ再生時、toioが記録通りに動き、音を再生]
```

## 機能

### Mode 1: ループシーケンサモード（DTM風）

複数のtoioを使って、それぞれ独立したループを作成・再生できるモードです。

#### 状態遷移

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   IDLE ──(ボタン)──→ RECORDING ──(ボタン)──→ WAITING            │
│  (白LED)    0.5秒後     (赤LED)              (黄LED)            │
│                                                 │               │
│                                            (ボタン)             │
│                                                 ↓               │
│   PAUSED ←──(ボタン)── PLAYING                                  │
│  (黄LED)              (緑LED)                                   │
│     │                                                           │
│     └──(ボタン)──→ RECORDING（再記録）                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

| 状態 | LED色 | 説明 |
|------|-------|------|
| IDLE | 白 | 初期状態。ボタンで記録開始 |
| RECORDING | 赤 | 動きを記録中 |
| WAITING | 黄 | 記録完了。ボタンで再生開始 |
| PLAYING | 緑 | ループ再生中 |
| PAUSED | 黄 | 一時停止中。ボタンで再記録 |

#### 波形の割り当て

各toioには異なる波形（音色）が割り当てられます：

| toio | 波形 | 発音方式 | 音の特徴 |
|------|------|----------|----------|
| toio1 | サイン波 | 位置に応じて常時発音 | 柔らかく純粋な音 |
| toio2 | のこぎり波 | 位置に応じて常時発音 | 明るく倍音豊富な音 |
| toio3 | 矩形波 | 磁石検知時のみ0.1秒間発音 | パーカッシブな音 |

### Mode 2: 重奏モード

2台のtoioを使い、片方の動きをもう片方が遅れて追従するモードです。

- **serve_toio**: ユーザーが手で動かすtoio
- **receive_toio**: serve_toioの動きを遅延追従するtoio
- デフォルト遅延: 2秒
- 両方のtoioで位置に応じた音が鳴る

### Mode 3: 動作確認モード

1台のtoioを使用して各機能をテストするデバッグモードです。

- 座標取得（リアルタイム表示）
- 磁石検知（リアルタイム表示）
- 座標移動（入力した座標へ移動）

### Mode 4: 保存データ再生モード

過去に保存した記録データを選択して再生するモードです。

- 保存されたJSONファイルを選択
- 各toioが記録通りに動き、音を再生

## アルゴリズムと実装詳細

### 1. 音声合成アルゴリズム

#### 座標から音への変換

toioの位置情報を音のパラメータに変換します。

```
音声合成マット領域:
  X: 135〜220 (音量軸)
  Y: 190〜315 (周波数軸)

┌─────────────────────┐
│ 高音量        高音   │ Y=315 (B5: 988Hz)
│   ↑            ↑    │
│  音量軸     周波数軸 │
│   ↓            ↓    │
│ 低音量        低音   │ Y=190 (C4: 262Hz)
└─────────────────────┘
 X=135              X=220
```

**変換式:**

```python
# Y座標 → 周波数 (線形補間)
frequency = (y - Y_MIN) / (Y_MAX - Y_MIN) * (FREQ_MAX - FREQ_MIN) + FREQ_MIN
# 範囲: 261.626Hz (C4) 〜 987.767Hz (B5)

# X座標 → 音量 (線形補間)
volume = (x - X_MIN) / (X_MAX - X_MIN) * 1.0
# 範囲: 0.0 〜 1.0
```

#### 波形生成

3種類の波形を位相連続で生成します：

```python
# サンプルレート: 44100Hz
# チャンクサイズ: 1024サンプル

# サイン波
samples = volume * sin(2π * frequency * t + phase)

# のこぎり波
samples = volume * (2 * ((phase / 2π) % 1) - 1)

# 矩形波
samples = volume * sign(sin(2π * frequency * t + phase))
```

**位相連続性の維持:**

音を途切れなく滑らかに再生するため、チャンク間で位相を連続的に保持します。

```python
# 次のチャンクの位相を計算
phase += 2π * frequency * chunk_size / sample_rate
phase %= 2π  # 0〜2πに正規化
```

### 2. 動き記録アルゴリズム

#### サンプリング

- **サンプリングレート**: 50Hz（20ms間隔）
- **記録データ**: X座標、Y座標、角度、タイムスタンプ、速度

#### 閾値ベースのフィルタリング

微小な振動やノイズを除去するため、有意な変化のみを記録します。

```python
# フィルタリング閾値
position_threshold = 5   # 位置: 5ピクセル以上の変化
angle_threshold = 10     # 角度: 10度以上の変化

# 有意な変化の判定
is_significant = (|dx| > 5) or (|dy| > 5) or (|dangle| > 10)
```

#### 速度の自動計算

再生時にtoioが記録通りに動けるよう、フレーム間の移動速度を計算します。

```python
# 距離を計算
distance = sqrt((x - last_x)² + (y - last_y)²)

# 速度を計算（ピクセル/秒）
velocity = distance / dt

# toio API用の速度にマッピング（50〜100）
speed = clamp(velocity / 2.5, 50, 100)
```

### 3. 位置ロスト時の処理

toioがマットから外れると位置検出ができなくなります。この場合の処理：

1. **音をミュート**: 位置検出できない間は無音にする
2. **記録時間を補正**: ロスト時間を記録開始時間に加算し、実質的な記録時間を正確に保つ

```python
if position_lost:
    # ロスト開始時刻を記録
    position_lost_time = current_time
    synth.mute()
else:
    if was_position_lost:
        # ロスト時間を記録開始時間に加算
        lost_duration = current_time - position_lost_time
        recorder.start_time += lost_duration
        synth.unmute()
```

### 4. 磁石検知トリガー（toio3専用）

toio3は通常の位置ベースの発音ではなく、磁石を検知した瞬間だけ音を発します。

```
検知フロー:
┌─────────┐     ┌─────────┐     ┌─────────┐
│ 磁石なし │ ──→ │ 磁石検知 │ ──→ │ 0.1秒後 │
│  (無音)  │     │ (発音開始)│    │ (無音)  │
└─────────┘     └─────────┘     └─────────┘
     ↑                               │
     └───────────────────────────────┘
```

**実装:**

```python
# 磁石検知の立ち上がりエッジを検出
if magnet_detected and not was_magnet_detected:
    magnet_sound_until = current_time + 0.1  # 0.1秒後に終了
    synth.unmute()

was_magnet_detected = magnet_detected

# 0.1秒経過したらミュート
if magnet_sound_until and current_time >= magnet_sound_until:
    synth.mute()
    magnet_sound_until = None
```

### 5. 再生時の移動制御

記録したフレームに従ってtoioを移動させます。

#### 移動の閾値フィルタリング

微小な移動指示を省略し、通信負荷を軽減します。

```python
PLAYBACK_POSITION_THRESHOLD = 8   # 位置: 8ピクセル
PLAYBACK_ANGLE_THRESHOLD = 15     # 角度: 15度

should_move = (|dx| > 8) or (|dy| > 8) or (|dangle| > 15)
```

#### タイミング制御

記録時のタイムスタンプに合わせて移動指示を送信します。

```python
for frame in frames:
    # 正確なタイミングまで待機
    while current_time - start_time < frame.timestamp:
        await sleep(0.005)  # 5ms精度で待機

    # 移動指示を送信
    await move_position(frame.x, frame.y, frame.angle, frame.speed)
```

│  │              BLE Communication Layer                 │  │

## 使い方

### 1. toioのMACアドレス設定

使用するtoioのBluetooth MACアドレスを `src/usecase/ui.py` に設定します：

```python
TOIO_ADDRESSES = [
    "e0:20:07:f8:6a:82",  # toio 1
    "e2:b2:40:be:b2:73",  # toio 2
    "ff:7e:ed:ba:75:86",  # toio 3
]
```

**MACアドレスの確認方法:**
- Windows: Bluetoothデバイスのプロパティから確認
- macOS: システム情報 > Bluetooth
- Linux: `bluetoothctl devices`

### 2. 実行

```bash
python src/main.py
```

### 3. モード選択

```
==================================================
toio 音楽プログラム
==================================================
モードを選択してください:
  1. ループシーケンサモード
  2. 重奏モード
  3. 動作確認モード
  4. 保存データ再生モード
  0. 終了
選択 (0-4):
```

### 4. 操作方法

#### Mode 1: ループシーケンサモード

1. toioをマット上に置く
2. toioのボタンを押す → 0.5秒後に記録開始（赤LED）
3. toioを手で動かして音を作成
4. もう一度ボタンを押す → 記録終了（黄LED）
5. ボタンを押す → ループ再生開始（緑LED）
6. 終了: `q` + Enter

#### Mode 4: 再生モード

1. 保存されたJSONファイルを選択
2. toioが自動的に動き、音を再生
3. 終了: `q` + Enter

## データ形式

### JSON記録フォーマット

```json
{
  "created_at": "2024-01-01T12:00:00",
  "toio_count": 3,
  "toios": [
    {
      "index": 0,
      "name": "toio_1",
      "wave_type": "sine",
      "frames": [
        {
          "x": 180,
          "y": 250,
          "angle": 45,
          "timestamp": 0.0,
          "speed": 100
        },
        {
          "x": 185,
          "y": 260,
          "angle": 50,
          "timestamp": 0.02,
          "speed": 75
        }
      ]
    }
  ]
}
```

| フィールド | 説明 |
|-----------|------|
| created_at | 記録日時（ISO 8601形式） |
| toio_count | 使用したtoioの台数 |
| index | toioのインデックス（0始まり） |
| name | toioの名前 |
| wave_type | 波形タイプ（sine/sawtooth/square） |
| x, y | マット上の座標 |
| angle | 向き（0〜359度） |
| timestamp | 記録開始からの経過時間（秒） |
| speed | 移動速度（50〜100） |
